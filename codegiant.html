BACKGROUD

Codegiant is a DevOps web-based tool designed for software development team, with issue tracking, hosted Git repositories, continuous integration and documentation and more all in one single platform. Focusing on how to streamline the developer experience and remove any friction in the development process. Developers should be able to code, deploy and manage their projects in one simple experience.

RESEARCH

For me, I didn’t have any coding background so I did a lot of research on the development livecycle and how each feature functions in the process of developing.
I researched some of the biggest competitors like Gitlab, Atlassian, JetBrians and other product management tools, CI/CD tools etc. I would sign up for the products and had decent experiences of how they’re designed this way, from my research I found that it’s common for the highly technical features to have very similar UIs and the biggest challenge in all would be how to keep the UI and the user journey as simple as possible since in most of the time there are lots of complex data and steps in one view.

FAST ITERATION

We worked as an agile team where I was the only designer, we would go over the user journey and make sure I understood the technical side of the feature, then I started designing.
Most of the time the design needs to be presented in a short period of time, the hand-off process was always abrupt. Multiple iterations can be expected after the UIs were in the development process, there were lots of overlaps for designing and developing in the same time so the faster and more precise the iterations can be, the smoother the development process is.

DESIGN DETAILS

Serverless workflows
Codegiant's serverless compute platform allows the users to run code without provisioning a server. Create, version, and run sequences of functions with the built-in serverless IDE.

Not only can the user write and run Functions-as-a-service (FaaS) with zero configuration, they can also view the metrics and test it in the serverless workflows.
